# Neural Service Mesh — استيراد المصفوفة الجديدة وتفعيلها بالكامل

قبل البدء بالمهمة:

1. سأرسل المصفوفة الجديدة في ملف منفصل بعد هذه الرسالة.
2. لا تبدأ أي تعديل قبل استلام الملف.
3. إذا احتجت GitHub Token أو صلاحية Push فاطلبها أولاً قبل تنفيذ المهمة.
4. بعد انتهاء التعديلات قم برفعها إلى GitHub مع Commit واضح.

━━━━━━━━━━━━━━━━━━

الهدف

استبدال الأوزان الحالية بالكامل بالمصفوفة الجديدة التي سأرسلها.

━━━━━━━━━━━━━━━━━━

المرحلة 1 — استلام المصفوفة

عند استلام الملف:

- اقرأ المصفوفة كما هي.
- لا تعدل أي قيمة.
- لا تنشئ أوزان عشوائية.
- لا تعيد التهيئة.
- لا تستخدم أي مصفوفة بديلة.

━━━━━━━━━━━━━━━━━━

المرحلة 2 — إزالة الاعتماد على الأوزان القديمة

إذا كان النظام ما زال يستخدم:

- shape=(9,7)
- أو checkpoint قديم

فيجب إيقاف ذلك واستبداله بالمصفوفة الجديدة فقط.

━━━━━━━━━━━━━━━━━━

المرحلة 3 — تفعيل Runtime

تأكد أن:

NeuralWeightLayer

يقوم بتحميل المصفوفة الجديدة فعلياً أثناء التشغيل.

المطلوب:

- shape = (108,7)
- bias = 0.6

وتكون هذه القيم هي المستخدمة فعلياً في التدريب والاستدلال.

━━━━━━━━━━━━━━━━━━

المرحلة 4 — الحفظ الدائم

احفظ المصفوفة في:

checkpoints/neural_weights.npy

واحفظ الانحياز في:

checkpoints/neural_bias.npy

━━━━━━━━━━━━━━━━━━

المرحلة 5 — التحقق من التشغيل

بعد الاستيراد:

- أعد تشغيل النظام.
- تأكد أن Runtime حمّل المصفوفة الجديدة.
- تأكد أن checkpoint يُقرأ بشكل صحيح.
- تأكد أن النظام لا يعود إلى المصفوفة القديمة.

━━━━━━━━━━━━━━━━━━

المرحلة 6 — تدريب فعلي

بعد تفعيل المصفوفة:

قم بتشغيل دورة تدريب حقيقية باستخدام:

- القرآن الكريم
- Cognitive Knowledge Graph (CKG)

إذا كانت متاحة.

الهدف:

التأكد أن التدريب يؤثر على الأوزان الجديدة فعلاً.

━━━━━━━━━━━━━━━━━━

المرحلة 7 — تقرير نهائي

أريد تقريراً يحتوي:

- shape
- total_weights
- bias
- training_steps_before
- training_steps_after
- mean_before
- mean_after
- std_before
- std_after
- min
- max
- checkpoint_exists
- checkpoint_path

━━━━━━━━━━━━━━━━━━

المرحلة 8 — إثبات الاستخدام

أريد دليلاً واضحاً أن:

- Runtime يستخدم المصفوفة الجديدة.
- التدريب تم فوقها.
- الحفظ نجح.
- إعادة التشغيل لا تعيد الأوزان القديمة.

━━━━━━━━━━━━━━━━━━

المرحلة 9 — GitHub

بعد نجاح كل الاختبارات:

- Commit واضح يشرح التعديلات.
- Push إلى GitHub.
- أعطني Hash آخر Commit.
- أعطني قائمة الملفات التي تم تعديلها.

لا تبدأ التنفيذ قبل استلام ملف المصفوفة الجديدة.
